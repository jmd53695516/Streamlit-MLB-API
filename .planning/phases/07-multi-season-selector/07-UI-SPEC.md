## Color

Streamlit's default light theme governs the palette. The 60/30/10 split below maps to Streamlit's semantic color usage.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (Streamlit default surface) | Page background, widget backgrounds |
| Secondary (30%) | `#F0F2F6` (Streamlit sidebar / card surface) | Expander backgrounds, `st.info` / `st.warning` containers |
| Accent (10%) | `#FF4B4B` (Streamlit primary red) | Spray chart: HRs that clear the selected park fence (green dot), and primary action elements |
| Destructive | `#FF4B4B` via `st.error` | Error banners only (MLB API failure) |

Accent reserved for:
- Spray chart HR dots that CLEAR the selected park fence (green: `rgba(44, 160, 44, 1.0)`)
- Spray chart HR dots that do NOT clear the selected park fence (red: `rgba(214, 39, 40, 1.0)`)
- `st.info` call-to-action text (Streamlit-managed)

Source: Existing `app.py` and `chart.py` color usage; Streamlit default theme. Do not add `.streamlit/config.toml` theme overrides in this phase (Phase 8 handles that).

Park Rankings table highlight colors (pre-established in v1.0):
- Top 3 parks: `rgba(44, 160, 44, 0.15)` green row background
- Bottom 3 parks: `rgba(214, 39, 40, 0.15)` red row background

## Widget Interaction Contract

### Selector Cascade Order

```
Season selectbox  →  Team selectbox  →  Player selectbox  →  Stadium selectbox
```

Source: CONTEXT.md D-01 (locked). Season is placed BEFORE Team in `app.py`.

### Season Selectbox

| Property | Value | Source |
|----------|-------|--------|
| Label | `"Season"` | D-01 |
| Widget | `st.selectbox` | Established pattern |
| Options | `AVAILABLE_SEASONS` from `config.py` (5 years descending, e.g. `[2026, 2025, 2024, 2023, 2022]`) | D-07 |
| Default | `index=0` (current year always first) | D-08 |
| Placeholder | none — a season must always be selected | D-08 + RESEARCH.md Pitfall 3 |
| `help` tooltip | `"Select a season to explore."` | default |
| `on_change` | `_on_season_change` | D-02 |
| `disabled` | never disabled — season is always available | D-01 |
| `key` | `"season"` | session state key |

### Cascade Reset on Season Change

When `_on_season_change` fires, ALL three downstream session state keys are set to `None`:

```
st.session_state["team_id"]   = None
st.session_state["player_id"] = None
st.session_state["venue_id"]  = None
```

Source: CONTEXT.md D-02 (locked — full cascade reset, not partial).

### Team / Player / Stadium Selectboxes (unchanged from v1.0)

No visual or interaction changes. These remain `index=None` with placeholders. The only change is that `team_id`-dependent calls now pass `st.session_state["season"]` instead of the `CURRENT_SEASON` constant.

### Player Label Format (unchanged)

`"{fullName} — {N} HR"` — includes season-specific HR count for the selected season.

### Disabled States (cascade enforcement)

| Widget | Disabled when |
|--------|---------------|
| Team | never (always populated from `get_teams()`) |
| Player | `team_id is None` |
| Stadium | `player_id is None` |

## States Checklist

| State | Trigger | UI Response |
|-------|---------|-------------|
| Cold start (no selections) | App load | `st.info("Select a team, player, and stadium to begin.")` |
| Season changed | User picks a season | `_on_season_change` nulls team/player/venue; Streamlit reruns; downstream selectors reset to placeholder |
| Team changed | User picks a team | `_on_team_change` nulls player/venue; player selectbox repopulates for that team + season |
| Player picked | User picks a player | `_on_player_change` sets venue to home park; stadium selectbox opens |
| All selected | team + player + venue all non-None | `st.spinner` → `build_view` → render spray chart + tables |
| Player has 0 HRs in season | `view.events` is empty | `st.info("{name} has no home runs in {season}.")` |
| Player has HRs but none plottable | `view.plottable_events` is empty | `st.info("{name} has no plottable HRs in {season}.")` |
| API failure | Exception in `build_view` | `st.error(...)` + `st.button("Retry")` + `st.stop()` |
| Partial feed failure | `view.errors` non-empty | `st.warning("{N} game feed(s) failed to fetch; HR data may be incomplete.")` |
