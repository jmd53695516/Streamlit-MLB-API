
**Step 3 — Push main to GitHub:**

```bash
git push -u origin main
```

**Step 4 — Verify push succeeded:**

Run `git log --oneline -3` on main to confirm the latest commits are present. Run `git branch -vv` to confirm main tracks origin/main.

**Deploy the app to Streamlit Community Cloud — step by step:**

1. **Go to** https://share.streamlit.io and sign in with your GitHub account
   - If you do not have a Streamlit Community Cloud account, click "Sign up" and authorize with GitHub OAuth

2. **Click "New app"** in the top-right corner of the dashboard

3. **Fill in the deploy form:**
   - **Repository:** Select your GitHub repo (e.g., `<username>/streamlit-mlb-hr-explorer`)
   - **Branch:** `main`
   - **Main file path:** `src/mlb_park/app.py` (this is NOT at the repo root — must enter the full path)

4. **Click "Advanced settings"** (optional but recommended):
   - **Python version:** Select `3.12` if available (matches local dev environment)
   - **Secrets:** Leave empty — no secrets needed (D-12, statsapi.mlb.com is public)

5. **Click "Deploy!"**

6. **Wait for build** (typically 2-5 minutes):
   - Cloud installs dependencies from `requirements.txt` (4 packages, no pytest)
   - Cloud reads `.streamlit/config.toml` for theme and server config
   - App starts with `src/mlb_park/app.py`

7. **Verify the deployed app:**
   - [ ] Page loads with wide layout (not narrow centered)
   - [ ] Browser tab says "MLB HR Park Explorer"
   - [ ] Theme has navy primary color (#002D72)
   - [ ] Season selectbox shows 2022-2026
   - [ ] Select a team -> player -> stadium flow works end-to-end
   - [ ] Spray chart renders with hover tooltips
   - [ ] No "fetching venue data" delay on cold start (venues_cache.json was committed)

8. **Copy the shareable URL** (format: `https://<app-name>.streamlit.app`)
